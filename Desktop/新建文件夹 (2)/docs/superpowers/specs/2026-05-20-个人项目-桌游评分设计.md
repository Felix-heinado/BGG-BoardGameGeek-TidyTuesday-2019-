# 个人项目：桌游复杂度与玩家评分的量化关系分析

## 概述

- **主题**：桌游复杂度与玩家评分的量化关系 — 基于 BoardGameGeek 数据
- **数据集**：BoardGameGeek（BGG）收录的桌游元数据，10w+ 条记录
- **数据类型**：结构化表格数据（高维表格 + 时序维度）
- **课程对齐**：教学大纲第三章「高维表格数据与特征工程」（PCA降维、聚类）、第四章「时间序列数据与行为建模」

## 数据说明

BGG 是全球最大的桌游数据库，每个游戏包含丰富的多维元数据。数据对应课件 Chapter3「六种数据类型」中的两类：

- **表格数据**：机制标签（100+类）、评分、复杂度权重、游戏时长、玩家人数、设计师、出版年份等（对应"高维表格与特征维度"）
- **序列数据**：评分随时间的演变、年发行量趋势（对应"序列数据与时间维度"）

## 核心研究问题

1. 桌游复杂度（Weight）与用户评分的关系是线性还是非线性（倒U型）？
2. 哪些机制（Mechanics）组合倾向产生高评分？能否构建"机制-评分"关联图谱？
3. 桌游行业的评分趋势如何演变？"桌游文艺复兴"是否存在统计证据？

## 作业1：阶段性探索（高维表格数据 + EDA）

### 步骤1：数据获取

- Kaggle BGG 数据集（课件 Chapter3 第34页推荐数据源）
- 备选：BGG XML API2 直接爬取（课件"在感兴趣的领域开发数据爬虫"）
- 数据收录范围：1950 年代至 2025 年的桌游记录

### 步骤2：数据清洗

严格对应课件 Chapter4「数据预处理」（第9-13页）方法论：

| 处理事项 | 方法 | 课件引用 |
|----------|------|----------|
| 冷门游戏过滤 | 评分人数 < 30 视为统计意义不足，剔除（Chapter4 第10页"缺失比例超过阈值"的扩展应用） | Chapter4 第10页 |
| 异常值检测 | 评分在 [1,10] 范围外的记录剔除；复杂度权重异常值用 Z-Score 标记（Chapter4 第11页） | Chapter4 第11页「Z-Score」 |
| 缺失机制/类别 | 分类字段缺失填充 "Unknown" 标签（Chapter4 第10页"UNKNOWN 填充"） | Chapter4 第10页 |
| 年份异常 | 出版年份晚于 2026 或早于 1900 视为错误数据剔除 | Chapter4 第9页「异常值」 |
| 机制解析 | 机制字段从 JSON/逗号分隔字符串拆分为 One-Hot 编码矩阵 | Chapter4 第17页「表格数据的特性」|

产出：清洗后数据集，字段完整、类型正确、统计可验证。

### 步骤3：EDA

遵循课件 Chapter3「从数据中获取知识/洞察」范式（第4-5页）：

1. **评分分布**：直方图 + KDE，判断是否正态分布，识别评分的集中趋势
2. **复杂度分布**：复杂度权重（1-5）的分布，轻策 vs 重策游戏占比
3. **年代趋势**：1970-2025 年每年桌游发行数量与平均评分折线图（对应"序列数据与时间维度"）
4. **Top-N 排行**：高评分游戏 Top20、高产设计师 Top10、最流行机制 Top15
5. **玩家人数分析**：最佳推荐人数分布，不同人数下的平均评分差异
6. **时长分析**：游戏时长分箱（<30min / 30-60 / 60-120 / 120+），各组评分差异

产出：6-8 张可视化图表，附业务解读。

### 步骤4：特征工程与相关性分析

对应课件 Chapter3「高维表格与特征维度」分析框架（第31页）及教学大纲"特征工程自动化筛选"：

1. **机制 One-Hot 编码**：100+ 机制标签转为特征矩阵
2. **复杂度-评分关系**：复杂度权重分段（1-1.5 / 1.5-2.5 / 2.5-3.5 / 3.5-5），箱线图对比
3. **时长与评分**：游戏时长 vs 平均评分散点图
4. **复合指标**：构建"机制多样性指数"（每款游戏拥有的机制数量）、"人气指数"（log(评分人数)）

### 步骤5：阶段性发现

对应教学大纲"研究能力"目标2——"对实验结果进行统计学处理，得出具有业务指导价值的工程结论"：

- 复杂度与评分的基本关系形态（初步判断：倒U型？线性？无显著关系？）
- 高分桌游的共性特征（机制组合、时长区间、玩家人数偏好）
- 2000年后桌游行业的"爆发式增长"趋势是否存在
- 提出作业2方向

## 作业2：深度建模与完整结项（新增内容）

作业2 在作业1 基础上，引入课程核心算法模块进行深度分析。

### 模块 A：PCA 降维 — 桌游机制空间可视化

**对应课件**：Chapter3「PCA降维技术」（高维表格数据 → 低维核心特征）

教学大纲明确要求"主成分分析降维技术"（教学要点第三章），以及"将数百个表层客观指标降维成少量稳定的潜在主成分"（Chapter3 第15页）。

**实现路径**：
1. 将 100+ 维的 One-Hot 机制矩阵标准化处理
2. 使用 PCA 降维至 2-3 个主成分，观察累计方差解释率
3. 在主成分空间中绘制每款桌游，按复杂度或评分着色
4. 分析各主成分对应的机制特征（哪些机制天然聚集）
5. 形成"桌游机制地图"——将复杂高维机制空间可视化呈现

**业务解释**：哪些机制属于同一"家族"？是否存在互斥的机制组合？

### 模块 B：聚类分析 — 桌游类型学构建

**对应课件**：Chapter3「聚类算法原理」及教学大纲"能对聚类结果进行深入的商业与经济学解释"

**实现路径**：
1. 基于机制特征 + 复杂度 + 时长做 K-Means 聚类（肘部法/轮廓系数确定 K 值）
2. 定义聚类标签（如：轻策聚会型、家庭亲子型、重策策略型、卡牌对战型）
3. 各聚类的典型特征描述 + 代表游戏列表
4. 对比聚类结果与 BGG 官方分类的一致性

**业务解释**：桌游行业的自然分类体系是什么？这如何帮助新游戏定位？（对应教学大纲"实现从海量数据点到精准业务洞察的逻辑闭环"）

### 模块 C：时间序列分析 — 桌游行业发展脉络

**对应课件**：Chapter4「时间序列数据与行为建模」及教学大纲"时序数据的平稳性检验、动态趋势分析与异常检测"

**实现路径**：
1. 构建年度时间序列：年均评分、年发行量、年均复杂度
2. 使用移动平均平滑波动（对应课件"移动平均趋势分析"）
3. 检测"桌游文艺复兴"的结构断点（对应课件"异常检测算法"）
4. 分析机制随时间演变的趋势（哪种机制在增长/衰落？）

**业务解释**：桌游行业是否存在代际更替的"黄金年代"？

### 模块 D：综合结论与结项报告

结合教学大纲"多维数据建模方法"和"复杂跨界问题解决能力"：

1. 复杂度与评分的最终量化模型（相关性 + 非线性检测 + 聚类验证）
2. "高分桌游配方"——机制组合 + 时长 + 复杂度 + 玩家人数的最优建议区间
3. 桌游行业发展趋势的历史叙事（从 1950s 到 2025）
4. 本研究的局限性与未来方向
